>
> 2026-07-16 v1 작성, 같은 날 **자기관리 앱으로 피벗** (Finch 모델 전면화) · 전략: Finch(코어 루프) × neal.fun(유입 엔진)
>
> **피벗 이유**: ① Finch가 증명한 지불의사 시장($30-40M ARR)이 웹에는 무주공산(Finch는 모바일 전용), ② 셀프케어 정체성 = 비게임 분류 → 앱인토스·스토어 입점이 가벼워짐(게임제작업·등급분류 회피), ③ 미니게임은 유입·보상으로 그대로 유효 — 버리는 것 없음.
>
> **정체성 문장**: "catnip은 고양이를 돌보는 게임이 아니라, 고양이가 나를 돌보게 하는 self-care companion이다."

## 1. 콘셉트

- **catnip = "사람을 위한 캣닢"**. 15초면 이해되는 고양이 미니게임들이 모인 놀이터.
- 모든 미니게임의 보상은 하나로 흐른다: **젤리(🐾 재화) → 내 고양이**.
- 설치 없음, 로그인 없음(초기), 브라우저에서 즉시. 영어·한국어 이중언어.
- 톤: 종이 질감 + 손으로 만지는 촉각. 지금 배포본의 아트 디렉션 유지.

## 2. 코어 루프 (피벗 후)

```
[코어 — 자기관리 루프, 매일]
오늘의 돌봄 체크인 (물 마시기·산책·스트레칭 등 내가 정한 1~3개)
      ↓ 체크 = 고양이 돌봄
   젤리 🐾 + 애정도 + 고양이 반응 (골골·하트)
      ↓
집중이 필요하면 /focus (고양이가 옆에서 잠) · 쉬고 싶으면 쓰다듬기·놀이
      ↓ 애정도 레벨업
액세서리·품종 언락 (수집) → "내가 나를 돌본 만큼 자란 고양이"
```

```
[유입 — 놀이 루프, neal.fun 엔진 유지]
미니게임 8종·퀴즈·월드컵 = 바이럴 티켓 + 젤리 보상 수단
공유 카드 → 신규 유입 → 입양 → 코어 루프 합류
```

- 미니게임은 제거하지 않는다 — "놀이 코너"로 위치만 바뀜 (Finch의 acorn 게임처럼 보상·휴식 장치).
- 카피 원칙: "게임하러 와" ❌ → **"오늘 하루를 돌보러 와, 고양이랑"** ⭕

## 3. 구조 — 허브 & 방

```
catnip.fun
├─ /            허브 = 내 고양이 방. 고양이가 앉아 있고(현 배포본),
│               쓰다듬기 가능. 주변에 미니게임 입구·간식 주기·꾸미기.
├─ /play/<game> 미니게임들 (아래 로스터)
├─ /cats        도감 — 품종·코트 수집 현황 (코숏 4종 → 확장)
└─ /focus       집중 타이머 (유틸리티 = "조용한 방")
```

- 허브가 곧 펫 화면. 미니게임천국의 "로비 = 캐릭터 성장 화면" 구조.
- 미입양 방문자: 허브에 데모 고양이 + "입양하기"만. 미니게임은 자유 플레이(젤리는 입양 후 적립).

## 4. 미니게임 로스터

공식: **촉각 인터랙션 1개 + 15초 안에 "우와" + 공유하고 싶은 순간 1개.**
기존 자산(스프링 필드·verlet 로프·인스턴싱 퍼·오버레이 얼굴)을 재활용해 제작비를 낮춘다.

## 5. 메타 프로그레션

- **입양 = 선택**: 첫 방문 시 4마리(치즈·고등어·삼색이·턱시도) 캐러셀에서 **한 마리를 골라 이름 짓기**(기본값 Nabi). 못 고른 품종은 도감에 실루엣으로 남아 수집 동기가 됨.
- **젤리 🐾** (재화): 미니게임 보상. 이름은 발바닥 젤리(toe beans)에서. ※"츄르"는 이나바 상표라 재화명으로 쓰지 않음.
- **애정도**: 젤리로 간식·놀이 → 애정도 경험치. 레벨 구간마다 언락.
- **수집 도감 = 품종이 핵심 축**: 프로시저럴이라 품종 추가 = 코트 함수 1개 + 상수 튜닝(아트 비용 ≈ 0). 로드맵: 4종(P0) → 검은고양이·흰고양이·러시안블루(P1, 애정도 언락) → 샴·장모종(털 길이 파라미터!)·시즌 한정(할로윈 검은냥 등)(P2+). 액세서리(리본·모자)·방 아이템(캣타워·쿠션)도 같은 축.
- **두 번째 고양이**: 높은 애정도 보상(또는 유료)으로 멀티펫 — P2+.
- **streak**: 매일 첫 방문 보너스 젤리 (가벼운 데일리 훅, 강요 없음).
- 저장: Phase 0-1은 localStorage. 계정(Supabase)은 리텐션 검증 후.

## 6. 유틸리티 통합 — /focus "조용한 방"

- Forest 모델: 타이머를 켜면 고양이가 옆에서 잠들고, 세션을 완주하면 **젤리 대량 보상**.
- 즉, 유틸리티도 미니게임 로스터의 하나("가장 보상이 큰 방")로 흡수 — 별도 제품이 아니라 같은 루프.
- 중도 이탈 시 고양이가 깸(벌점 없음, 죄책감 디자인은 배제 — 힐링 톤 유지).
- 후순위(P1): P0에서는 방만 만들어두고 "준비 중" 표시로 수요 신호만 측정해도 됨.

## 7. i18n

- 영어 기본, 한국어 동시 지원. `?lang=` + navigator.language 자동.
- 텍스트가 극히 적은 제품(버튼·이름·공유 카드)이라 JSON 사전 하나로 충분.
- 고양이 이름 기본값: **Nabi** — 입양 플로우에서 "한국에서는 고양이를 나비라고 불렀어요" 한 줄 스토리.

## 8. 기술 방침

- **정적 사이트 유지** (빌드 없음 → 필요해지면 Vite). 페이지당 HTML 1장, 공유 엔진은 ES 모듈로 분리:
  - `engine/fur.js` (인스턴싱 퍼 + 스프링 필드) · `engine/chain.js` (verlet) · `engine/cat.js` (품종·코트·얼굴) · `engine/save.js` (localStorage 스키마 + 버전) · `engine/i18n.js`
- 사운드: Web Audio 합성(골골·야옹) — 에셋 0.
- 분석: Vercel Analytics + 커스텀 이벤트 몇 개(입양, 게임 시작/완료, 공유).
- 저장 스키마에 버전 필드 필수 (나중에 마이그레이션).

## 13. SEO / GEO

**전제 수정 필요**: 현재 사이트는 텍스트가 거의 없어 크롤러·LLM에 안 잡힘. 캔버스 아래에 크롤러블 텍스트 레이어(About·FAQ·게임 설명)를 깐다 — 사람 눈엔 미니멀, 봇에겐 풍부하게.

**SEO**
- **타깃 쿼리**: "virtual cat", "pet a cat online", "online cat games free", "virtual pet browser", "고양이 키우기 게임", "고양이 게임" — 게임 페이지마다 쿼리 1개씩 매핑해 타이틀·H1·설명 작성
- **프로그래매틱 SEO — 도감이 콘텐츠 자산**: `/cats/korean-shorthair`, `/cats/calico` 등 품종 페이지에 품종 정보(성격·특징) + 인터랙티브 데모 임베드. "korean shorthair", "calico cat" 류 검색 유입 → 입양 퍼널. 품종이 늘수록 SEO 표면적도 늘어나는 구조
- **기술 SEO**: 정적 HTML이라 이미 유리. 페이지별 메타·OG 이미지·sitemap.xml·JSON-LD(VideoGame/WebApplication 스키마)·hreflang(en/ko). 초경량 페이지 = Core Web Vitals 만점권
